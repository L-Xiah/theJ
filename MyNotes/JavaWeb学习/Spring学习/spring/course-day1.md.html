Spring介绍

Spring是一个开源框架，Spring是于2003 年兴起的一个轻量级的Java 开发框架，由Rod Johnson创建。简单来说，Spring是一个分层的JavaSE/EE full-stack(一站式) 轻量级开源框架。 Spring是全面的和模块化的。Spring有分层的体系结构，这意味着你能选择使用它孤立的任何部分，它的架构仍然是内在稳定的。因此从你的学习中，你可得到最大的价值。例如，你可能选择仅仅使用Spring来简单化JDBC的使用，或用来管理所有的业务对象。

Spring能提供的功能：

  • 容器
  • 提供对多种技术的支持(JMS, MQ, UT等)
  • AOP（事务管理、日志等）
  • 提供了多方应用api的工具(JDBC template等)
  • 对主流框架的支持(Hibernate, Struts等)

官方网站：http://spring.io/

IOC控制反转

IoC(Inversion of control) - 控制反转。其并不是一种具体的接口、技术，而是一种编程思想。通过使用控制反转，将程序中设计好的对象，通过某种方式（如接口注入等方式）交给容器，让容器控制这些对象的实例化、销毁等控制。

  • 控制的概念：按照一般Java的思想，在编程的时候会主动new一个对象并且操作这个对象，这时就是其他的程序逻辑需要主动创建并依赖这个对象；而使用IoC容器将这一动作反转，程序只需要关心对象的调用、操作，而不用关心它是“从哪里来”的。这也符合面向接口编程的思想，同样，这一思想并不局限于对象实例，也包括外部资源读取（如文件）等。
  • 反转-正转：与反转对应，普通地通过主动地方式获取依赖对象是“正转”；但由容器辅助对对象进行查找、依赖注入，对象被动地进行接受依赖关系，这一操作即是反转操作。

DI(Dependency Injection) - 依赖注入。将组件之间的关系让容器在运行的时间内决定。由容器动态地将依赖关系注入到组件之中。通过依赖注入的方式，只需要进行一些（简单的）配置，无需任何代码就可以指定目标需要的资源，完成自身业务逻辑，而不用关心资源的来源。

Bean配置

作用域
类型 说明
singleton 单例，bean容器只存在一份实例（默认）
prototype 每次请求（使用）创建新的实例，destory方式不生效
request 每次http请求时创建实例且仅在当前请求内生效
session 同request，当前session内生效
global session 基于portlet的web中生效（一般情况下同session） 
<bean ... scope="xxx"></bean>

生命周期

  • 定义
  • 初始化: 通过context读取xml文件加载
  • 使用: 调用bean实例
  • 销毁: bean容器对已经不使用的bean进行销毁（GC等）

初始化：

  • 实现org.springframework.beans.factory.InitializingBean接口，实现afterPropertiesSet()方法
  • 配置init-method
<!-- xml -->
<bean ... init-method="xxx"></bean>
 
// xx.java
public void init() {
  // init here
}

销毁：

  • 实现org.springframework.beans.factory.DisposableBean接口，实现destory()方法
  • 配置destory-method
    code与初始化基本相同

全局定义

<?xml ...?>
<beans ... default-init-method="init" default-destory-method="destory">
  ...
</beans>

Bean相关注解例子

Spring 3.0以后引入了注解机制，不再使用xml定义bean的相关内容，而是使用java注解的方式进行bean特性的声明（当然xml仍然兼容）。Spring本身会“扫描”Bean的关系，并将其注册到ApplicationContext中。

xml配置
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:context="http://www.springframework.org/schema/context"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd" >
 
        <!-- 与单个context相关 -->
        <context:component-scan base-package="需要检测的package"></context:component-scan>
 
        <!-- context:annotation-config是包含在component-scan中的 -->
 </beans>
常用注解

类注解 - 使用类注解对bean直接进行生成，与在xml中定义相同，默认情况下会生成与类名相同但首字母小写的bean id

  • @Component 通用注解（元注解）
  • @Repository 通常使用在DAO类，持久层
  • @Service 通常注解在Service类，服务层
  • @Controller 通常用于Controller类，控制层
<!-- 也可以通过自定义方式定义命名规则 -->
<beans>
  <context:component-scan base-package="xxx" name-generator="xx.xx.MyNameGenerator" />
</beans>
 
// MyNameGenerator.java
public class MyNameGenerator implements BeanNameGenerator {
  // 实现BeanNameGenerator接口
}
  • @Scope 与xml中定义的作用域相同singleton, prototype等
<!-- 自定义scope -->
scope-resovler="xx.xx.MyScopeResovler"
 
// MyScopeResovler.java
public class MyScopeResovler implements ScopeMetadataResolver {
  // 实现ScopeMetadataResolver接口
}

  • @Required 定义域bean属性中的setXxx()方法，表示该属性必须在配置时被填充，通过bean定义或装配一个明确的属性值

  • @Autowired 与xml的default-autowire相同，自动装配。可声明在变量、set方法和构造函数上

    public class Student {
    @Autowired
    private Address address;
     
    // @Autowired(required = false)
    // 在使用构造函数的@Autowired时，注意只有一个构造函数可以标明required(此时建议使用@Required)
    }

    同时，Spring中内置的一些接口（类）如ApplicationContext, BeanFactory, Environment, ResourceLoader等都可以直接通过@Autowired注解进行装配，这样就不用关心实现和初始化过程，直接调用就可以了。
    集合的使用在@Autowired也是可以的。如果集合装配需要有序排列，则添加@Order接口。

  • List
  • Map - 以bean id作为key

@Autowired本身是通过BeanPostProcessor来处理的，所以不能在自己实现的BeanPostProcessor或BeanFactoryPostProcessor类中添加这些注解，如果有类似的需求，只能通过xml或者@Bean的方式进行注解

  • @Qualifier 自动装配的时候可能出现多个bean的实例情况，可以通过使用@Qualifier来限定范围（指定唯一），可以指定单独的构造器或者方法，也可以注解集合变量
@Autowired
@Qualifier("infosys") // 指定为infosys的bean
private Address address;
 
@Autowired
public void prepare(@Qualifier("infosys")Address address, XX xx) {
  // 指定第一个参数address为必须是infosys的bean
  ...
}
 
<!-- xml中实现qualifier -->
<bean class="xx.xx.Xxx">
  <qualifier value="infosys" />
</bean>

另有一个注解是@Resource(JSR-250标准中的注解)与@Qualifier类似，在使用map或其他集合的时候，由于语义上的差异，这些bean无法通过@Autowired注入，此时使用@Resource注解，通过唯一名称引用Map或集合的bean。@Resource适用于变量、set方法等，在目标是构造函数或多参方法时，最好使用@Qualifier配合@Autowired。

  • @Bean 标识一个用于配置和初始化IoC容器管理新对象的方法，类似于<bean/>

  • @Configuration 标识配置方式

    @Configuration
    public class SpringConfig {
      @Bean
      public UserService UserService() {
          return new UserServiceImpl();
      }
     
      // @Bean(name = "xxx")
      // @Bean(initMethod = "init") // 定义初始化方法
      // @Bean(destoryMethod = "destory") // 定义销毁方法
    }

@Bean和@Scope同样可以通过声明Bean的作用域

  • @Resource, @PostConstruct和@PreDestory 是JSR-250中的注解

    • @Resource 主要用于注解变量和setter方法，默认name属性是变量名

      注解提供的名字(name)被解析为bean的名称，是由ApplicationContext中的CommonAnnotationBeanPostProcessor处理的

    • @PostConstruct 初始化回调

    • @PreDestory 销毁前回调

      要使用这些JSR注解，必须确认CommonAnnotationBeanPostProcessor在Spring的ApplicationContext中注册

      另还有JSR-330的注解，需要通过maven引入javax.inject包，包含@Inject, @Named等注解
