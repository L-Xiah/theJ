Hibernate 入门

* Hibernate注解

Hibernate注解的目的

提供了一套基于Java Annotation的关键字进行注入式开发，简化繁琐的映射文件xml配置。在底层框架实现的效果是一样的。

JPA

JPA - Java Persistence API(Java持久化API接口)，是基于J2EE的标准化接口。而Hibernate实现了该接口，但是功能上是JPA的超集（包含）。主要通过hibernate-annotation, hibernate-entitymanager, hibernate-core来实现的这些功能。

开发的时候优先考虑JPA注解，这样能更方便程序的扩展和移植。

注解种类
  • 类级别注解
  • 属性级别注解
  • 映射关系注解

使用类注解需要在hibernate.cfg.xml中添加<mapping class="xx.xx.Class" />

@Entity类注解

映射的实体类 - 对应数据库的某张表

@Entity // 默认，则表名和实体类名相同
@Entity(name = "tableName") // 指定表名

使用Entity的时候，必须指定表的主键id，否则表无法创建

@Table类属性注解

通常和@Entity配合使用，标注在实体类的class定义处，表示实体类定义映射的表信息。

@Table(name="", catalog="", schema="")

DataBase - Catalog - Schema

不同数据库对Schema和Catalog的实现是有很大差别的(Oracle不支持catalog属性，而schema属性默认是用户名)

@Embeddable类注解

作为类属性嵌入到其他类中 - 组件属性

@Embeddable
public class Address {}

@Id属性注解

ID注解是必须的，定义了数据库中表的主键属性。可以存在联合主键（多个@Id，如果定义联合主键，必须实现Serializable），该注解放在主键属性或getXxx()之前

@GeneratedValue

主键生成策略

  • strategy=GenerationType
    • GenerationType.AUTO - 由底层数据库自动选择
    • GenerationType.IDENTITY - 根据数据库的identity生成
    • GenerationType.SEQUENCE - 序列生成
    • GenerationType.TABLE - 使用指定表来生成，配合@TableGenerator使用
      @Id
      @TableGenerator(name = "table_student", allocationSize = 1)
      @GeneratedValue(strategy = GenerationType.TABLE)
      如果需要实现assigned即外部赋值时，应有如下写法：
      @Id
      @GeneratedValue(generator = "sid")
      @GenericGenerator(name = "sid", strategy = "assigned")
      private String sid;
      Oralce中如果想使用Sequence，需要指定以下属性：
      @GeneratedValue(
              strategy = GenerationType.SEQUENCE,
              generator = "xxsequence")
      @SequenceGenerator(
              name = "xxsequence",
              sequenceName = "xx_sequence",
              allocationSize = 1,
              initialValue = 1)
      // id的增长是按照hibernate的增长决定的，而非数据库中定义的sequance，所以必须增加后两项属性

@Column属性注解

用来把属性映射到表中的列后，一些详细的定义（长度、默认值、空/非空等）

  • name - 数据库中字段名，默认与变量名一致
  • nullable - 是否可以为空，默认true
  • unique - 是否唯一，默认false
  • length - 字段的大小，仅对String类型生效，默认255
  • insertable - 在执行insert操作时，该字段是否可以被插入，默认true
  • updateable - 同insert，更新时是否更新该字段，默认true

@Embedded属性注解

与@Embeddable对应，表示嵌入的属性

@Embedded
@AttributeOverrides(
  {
    @AttributeOverride(
      name="city",
      column=@Column(
        name = "home_city",
        length = 25,
        nullable = false)),
    @AttributeOverride(
      name="postcode",
      column = @Column(
        name = "home_postcode",
        length = 15))
  }
)
private Address address;

@EmbeddedId属性注解

通常是用来实现嵌入式复合主键，有以下要求：

  • 满足JavaBean(属性私有，无参构造)
  • 重写equals()和hashCode()方法
  • 实现Serializable接口
// StudentPK.java - 一个新的主键类，包含学生id和身份证号作为复合主键
@Embeddable
public class StudentPK implements Serializable {
  // 生成SerializableID ...
 
  private String sid;
  private String identId,
 
  // 无参构造 ...
 
  // 重写hashCode() ...
  // 重写equals() ...
}
 
 
// Student.java
@EmbeddedId
private StudentPK pk;

@Transient属性注解

可选描述，如果添加此注解，则默认情况下，ORM框架会将此属性忽略掉，即如果这个属性不需要添加（映射）到数据库表中，将其注解为@Transient，则创建表会忽略这个字段

默认属性都是@Basic

关系型注解

与one-to-one, one-to-many, many-to-one, many-to-many在xml中的定义相同，注解也可以实现这些声明

一对一的单向外键关联

以下注解声明在主表中：

  • @OneToOne(cascade = CascadeType.ALL)
  • @JoinColumn(name = "pid", unique = true)
    • name指的是关联表的外键
    • unique表示唯一

      保存时应该先保存外键对象，然后再保存主表对象

// Student.java
@OneToOne(cascade = CascadeType.ALL)
@JoinColumn(name = "pid", unique = true)
private IDCard card;
 
// IDCard.java
@Entity
public class IDCard {
  @Id
  @GeneratedValue(generator = "pid")
  @GenericGenerator(name = "pid", strategy = "assigned")
  @Column(length = 18)
  private String pid;
 
  private String sname;
}

一对一双向外键

  • 主控方按照标准配置（注解）
  • 被控方添加@OneToOne(mappedBy = "card")

    双向关联（外键）的时候，必须设置mappedBy属性，只能交给一方去控制，如果双方都进行控制的话，会产生递归调用，无法保存

// IDCard.java
@OneToOne(mappedBy = "card") // 设置为交给属性的属性名
private Student stu;

多对一单向外键关联

添加如下注解：

  • @ManyToOne(cascade = {CascadeType.ALL}, fetch = FetchType.EAGER)
    • cascade - 级联
    • fetch - 抓取策略，EAGER, LAZY等
  • @JoinColumn(name = "tid", referencedColumnName = "tid")
    • name - 多方对应一方的成员变量名
    • referencedColumnName - 参照id，多方会根据这个生成列

      referencedColumnName这个不写的话，会自动根据name变量名去找

// Student.java
@ManyToOne(cascade = {CascadeType.ALL}, fetch = FetchType.EAGER)
@JoinColumn(name = "tid", referencedColumnName = "tid") // 外键
private Teacher teacher;

一对多外键关联

添加如下注解：

  • @OneToMany(cascade = {CascadeType.ALL}, fetch = FetchType.LAZY)
  • @JoinColumn(name = "tid")

一对多（多对一）的双向关系

多方：持有一方的引用

@ManyToOne(cascade = {CascadeType.ALL}, fetch = FetchType.EAGER)
@JoinColumn(name = "tid")

一方：持有多方的集合

@OneToMany(cascade = {CascadeType.ALL}, fetch = FetchType.LAZY)
@JoinColumn(name = "tid")

mappedBy与之前xml中的inverse属性是相同的，在这里可以写成mappedBy = "teacher"，对应的是需要在Student中的teacher对象维护，也就是把这个inverse到Student类中

多对多单向关系

在xml配置文件的many-to-many方法中，为了维护多对多的关系，需要添加一张新表，来标识所有的对应关系，所以需要添加以下内容：

// Student.java
@ManyToMany
@JoinTable( // 维护及生成中间表
    name = "relation_stules", // 中间表名
    joinColumns = {@JoinColumn(name = "sid")}, // 对于student的外键（字段）
    inverseJoinColumns = {@JoinColumn(name = "lid")} // 反转控制字段到lesson的lid
)
private Set<Lesson> lessons;

多对多的双向关系

与xml中定义many-to-many基本类似，只要通过mappedBy设置主控方就可以了

// Lesson.java
@ManyToMany(mappedBy = "lessons")
 
// Student.java
@ManyToMany
@JoinTable(
    name = "relation_stules",
    joinColumns = {@JoinColumn(name = "sid")},
    inverseJoinColumns = {@JoinColumn(name = "lid")}
)

对应关系总结：

  • 一对一单向：@OneToOne
  • 一对一双向：@OneToOne(mappedBy = "xxx")
  • 多对一单向：
    • @ManyToOne
    • @JoinColumn
  • 一对多单向：
    • @OneToMany
    • @JoinColumn
  • 一对多双向：
    • @ManyToOne
    • @OneToMany
    • @JoinColumn
  • 多对多单向：
    • @ManyToMany
    • @JoinTable
  • 多对多双向：
    • @ManyToMany(mappedBy = "xxx")
    • @ManyToMany
    • @JoinTable
